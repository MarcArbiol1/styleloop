# StyleLoop

**Upload an outfit photo → StyleLoop detects each garment and finds visually similar
items from a 39,000-product catalog** using a vision-language model for detection and
fashion-tuned image embeddings for similarity search.

It's an honest visual-similarity engine, not a magic "find the exact item to buy"
button — see [What works / what's hard](#what-works--whats-genuinely-hard) below.

## What it does

- **Outfit breakdown** — upload a photo and Google **Gemini 2.5 Flash** detects each
  clothing item: category, color, fit, material, and a bounding box per garment.
- **Visual similarity search** — crops a chosen garment from the photo, embeds it with
  **Marqo-FashionSigLIP** (768-dim, fashion-tuned), and finds the nearest products in a
  **SQLite** vector store via cosine similarity, with a category filter, color re-rank,
  and a similarity floor so it returns nothing rather than junk.
- **Wardrobe prototype (browser-only)** — closet building, OOTD capture, occasion-based
  outfit suggestions, cost-per-wear insights, live weather (Open-Meteo) and `.ics`
  calendar context. This layer is a local-storage prototype around the two AI features.

## How it works

```
Browser app (127.0.0.1:8001)
│
├─ "Break down the fit"  ──▶  Gemini 2.5 Flash (cloud, your API key)
│                              returns detected items + bounding boxes as JSON
│
└─ "Find similar"        ──▶  Local Python backend (localhost:8008)
                               crop to garment box ─▶ FashionSigLIP embed
                               ─▶ cosine search over SQLite catalog ─▶ nearest products
```

- **Gemini** does detection/description only — a VLM can't return real shoppable
  products by itself.
- **FashionSigLIP + SQLite** is the actual visual-similarity engine. Quality scales with
  the catalog.

## Tech stack

Vanilla JS / HTML / CSS frontend (no build step) · Python + Flask backend ·
Marqo-FashionSigLIP via `open_clip` · PyTorch (Apple Silicon `mps`) · SQLite + NumPy
cosine search · Gemini 2.5 Flash for detection · catalog from the HuggingFace dataset
[`ashraq/fashion-product-images-small`](https://huggingface.co/datasets/ashraq/fashion-product-images-small).

## Quick start

You need **two servers** running.

### 1. Frontend
```bash
python3 -m http.server 8001 --bind 127.0.0.1
# then open http://127.0.0.1:8001
```
On first analysis the app asks for a **Gemini API key** (free tier from
[Google AI Studio](https://aistudio.google.com/apikey)). It's stored only in your
browser's `localStorage` — never committed.

### 2. Backend (visual search)
```bash
cd backend
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# Build the catalog + vector index (one-time; downloads the dataset, ~12 min on a Mac GPU)
python load_subset.py 44000 999999   # extract images + catalog.json
python build_index.py                # embed → catalog.db

python server.py                     # serves http://localhost:8008
```
Health check: <http://localhost:8008/health> → `{"ok": true, "indexed": 39093}`.

> The catalog images (~153 MB), the built index `catalog.db` (~160 MB), and
> `catalog.json` (~14 MB) are **not** committed — they're regenerated by the two
> scripts above. That keeps the repo small and reproducible from source.

## What works / what's genuinely hard

**Works well:** on reasonably clean garment shots the similarity engine is solid —
a red-heels query returns red heels, a blazer photo ranks the blazer first. Detection
via Gemini is accurate for category, color, and fit.

**Genuinely hard (and honestly out of scope for a hobby budget):**
- *Exact product match from an arbitrary social photo.* No system does this well without
  Google Lens-tier infrastructure. StyleLoop aims for **visual similarity**, not exact match.
- *Live buy links.* The research dataset has none, so buy links are Google Shopping
  searches of the product name. Real shoppable links would need a paid product API (e.g.
  SerpAPI / Google Lens).
- *Messy photos.* Heavy occlusion, odd angles, and tiny accessories degrade both
  detection and cropping.

## Limitations & notes

- Gemini free tier is rate-limited (~20 req/min) — space out clicks or enable billing.
- The wardrobe/outfit-generation layer is a local-storage prototype with heuristic
  ranking, not a production backend.
- The Gemini key is sent directly from the browser — fine for local use, but a public
  deployment should proxy it through a server.
